# Тестування жадібного алгоритму та дінамічного програмування

## Тестування:

В цьому коді реалізовано тестування двох алгоритмів:

- Жадібний алгоритм
- Алгоритм дінамічного програмування

На основі двох різних алгоритмів написані функції для набору найменшої кількості монет видічі решти покупцеві

- Набір потрібноії суми відбувається такими номіналами монет: [50, 25, 10, 5, 2, 1]
- Тестування проводилось на трьох сумах різної величини : [11, 113, 58244]
- Для виміру часу використовувався timeit.
- Для вміру обсягу задіяної пам'яті використовувався tracemalloc.

## Результати:

1. **Швидшим алгоритмом, як і очікувалося, виявився жадібний алгоритм**.

- Різниція в швидкості геометрично зростала при рості розраховуємої суми:

  - для 11 центів жадібний алгоритм підбере монети у 8.2 разів швидше
  - для 113 - у 51.9 разів швидше
  - для 58244 - у 237.1 разів

2. **Жадібний алгоритмм використовув також менше пам'яті на піці**

- Чим більша сума решти тим більша різниця в пам'яті:

  - для 11 центів жадібний алгоритм використовував у 1.7 менше пам'яті
  - для 113 - у 9.5 разів швидше
  - для 58244 - у 7864 разів

3. **У даному випадку жадібний алгоритм показує такий самий результат, як і динамічний**. (Але це не завжди так.) Причина такого якісного результату жадібного алгоритму у найменшому поділі номіалів монет до 1 цента.

## Висновок:

- Жадібний алгоритм працює швидше динамічного і використовує менше пам'ті особливо при наборі великих сум.
- Якщо якість підбору не критична, доцільно використовувати жадібний алгоритм
